# Market Customer Order Analysis (SQL)

## Project Overview
This project focuses on analyzing customer purchasing behavior and product performance using SQL.  
The analysis is performed on a market dataset containing orders, products, aisles, and departments to uncover meaningful business insights that can support decision-making.

The goal of this project is to demonstrate practical SQL skills by solving real-world business questions related to customer behavior, product demand, and departmental performance.

---

## Dataset Description
The dataset used in this project consists of the following tables:

- **Orders** – Contains customer order details
- **Products** – Includes product-level information
- **Aisles** – Categorizes products into aisles
- **Departments** – Higher-level classification of products

These tables are joined to perform comprehensive analysis across different dimensions.

---

## Business Questions Answered
- Which departments generate the highest number of orders?
- What are the top-performing products based on order frequency?
- Which aisles contribute most to overall sales volume?
- How frequently do customers reorder products?
- What insights can be derived about customer purchasing patterns?

---

## SQL Concepts Used
- INNER JOIN & LEFT JOIN
- GROUP BY and HAVING clauses
- Aggregate functions (COUNT, SUM, AVG)
- Subqueries
- Window functions
- CASE statements
- Sorting and filtering using ORDER BY and WHERE

---

## Key Insights
- Identified high-demand departments and aisles to support inventory planning
- Analyzed top-selling products based on customer order frequency
- Observed customer reordering behavior to understand product loyalty
- Highlighted areas where business performance can be optimized
